# Antariksh Archive

A place for hackathon projects that didn't win, but were real work.

## Why this exists

A hackathon judge sees a demo, a pitch, and a leaderboard number. They don't
see the 100GB dataset you learned to parse in a weekend, the statistical
method you picked to avoid overclaiming on 51 real events, or the tradeoff
you made on purpose because the honest answer mattered more than the flashy
one. Most of that never gets written down anywhere. It just disappears when
the hackathon ends.

This repo is where it doesn't disappear.

**This is not a "we didn't win but we're actually great" repo.** Some
projects here placed, most didn't, and that's not the point either way. The
point is documenting real work clearly enough that someone else — a teammate,
a future you, a stranger building on the same problem — can actually learn
something from it. A well-documented project that placed 40th is more useful
here than a vague one-liner about a project that won.

## What belongs here

- Any hackathon project you built, with a real writeup — what you built, why,
  what you're proud of, what you'd fix
- Projects with partial results, non-working demos, or "we ran out of time"
  endings — document what you learned anyway
- Projects across any hackathon, not just BAH — this started there but isn't
  limited to it

## What doesn't

- Pure marketing copy with no technical substance
- Someone else's project without their consent to include it
- Anything you can't honestly stand behind if someone asks a hard question
  about it

## How to add your project

See [CONTRIBUTING.md](./CONTRIBUTING.md). Short version: copy the
[template](./TEMPLATE.md), fill it in, open a PR into `projects/your-project-name/`.

## Structure

```
projects/
  your-project-name/
    README.md      ← filled-out template
    (anything else you want to include — screenshots, links, etc.)
```

See [`projects/_example-entry/`](./projects/_example-entry/) for a filled-out
example.

## Maintenance

PRs are reviewed for completeness (does it follow the template, is it
someone's real project), not for how "good" the project was. Review target
is within a few days — if it's been longer, ping in the Discord(sharmaji0791)
